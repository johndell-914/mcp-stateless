The scenario — agentic commerce is real

Shopify, Stripe & others already ship remote MCP servers where an AI assistant builds a cart across a multi-turn conversation. This demo runs exactly that: three tools — create_cart, add_item, get_cart — the same ones a real commerce MCP server exposes. The tools were never the problem; the protocol was.

The catch — every conversation is a session

Under today's protocol the server mints an Mcp-Session-Id on initialize, and that session lives in one instance's memory. Put two instances behind a plain load balancer and the agent's next call can miss its session — “Session not found.” To cope you bolt on sticky routing and a shared session store; when a pod recycles, live agents drop mid-task. That is the operational tax.

The fix — MCP goes stateless (2026-07-28)

The release candidate removes the handshake and the session id. State moves into an explicit cart_token the client carries, so any request runs on any instance behind a plain round-robin load balancer. Delete the sticky gateway, delete the store — and autoscale freely.
