mcp-stateless-demo Code Architecture

A live before/after demo of MCP going stateless at the protocol layer · MCP 2.0 · Starlette · Gradio · Cloud Run · Supabase
🚪 Entry points
🖥️
Gradio UI
ui/gradio_app.py + panels.py · the live demo, driven in the browser
⌨️
CLI runner
client/__main__.py · python -m …client [legacy|auto] [--blast N]
⚙ src/mcp_stateless_demo

CONFIG
config.py
Settings (pydantic-settings) · one typed source of env
One place holds every setting for the demo, read from the environment. Change a value here and every service picks it up.
env
⚖️
LOAD BALANCER
proxy/app.py
round-robin · learned sticky · /config /target /kill /revive
The traffic director. It sends each request to one of the servers, and can be told to knock a server offline to show what breaks.
LB
🖥️
MCP SERVER
server/app.py · tools.py
MCPServer · streamable-http · stateless_http flag · 3 @server.tool()
The actual server the assistant talks to. It offers three actions and runs them. One switch on it decides whether it must remember conversations or not.
3 tools
explicit-handle pattern
🎫
DOMAIN
cart/ · token.py · store_postgres.py
signed cart_token handle + Postgres store
✓ Solves: no server-side session — any instance serves any request
Holds the cart logic: a tamper-proof claim-ticket the client carries, plus where the cart is actually saved.
domain
📊
OBSERVABILITY
cloud/logs.py
reads real Cloud Run stdout · LogProof
Pulls the platform's own logs so the demo can prove, with receipts, that the requests really happened across many machines.
logs
one tool call, entry → database
📤
1 · CLIENT
client/runner.py
ActRunner · Client(url, mode) · run_act / run_blast
Sends the assistant's requests. It opens a connection and calls an action by name.
client
⚖️
2 · ROUTE
proxy/app.py · forward()
pick(session_id) → forward raw bytes
Chooses which server machine handles the request and passes it along untouched.
route
🔧
3 · TOOL
server/tools.py
@server.tool() create_cart / add_item / get_cart
The action itself runs here — create a cart, add an item, or read the cart back.
@tool
state rides in the token
🎫
4 · HANDLE
cart/token.py
encode = id.HMAC · decode verifies the signature
✓ Solves: state travels in the request, not server memory
Turns the cart into a tamper-proof claim-ticket, and checks the ticket is genuine on the way back.
HMAC
🐘
5 · STORE
cart/store_postgres.py
create → INSERT · add_item → items || jsonb
Saves and updates the cart in the real database.
asyncpg
proving it with the platform's own logs

ui/gradio_app.py · _pull_logs()
retries past ingestion lag · ↻ Refresh re-pulls
After a run, the demo asks for the platform's logs, retrying for a few seconds until they show up.
retry

read-only
